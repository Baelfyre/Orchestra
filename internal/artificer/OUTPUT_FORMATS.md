# Artificer Output Formats

Artificer generates two distinct outputs: **Individual Source Audits** (per repository analysed) and the **Amalgamated Orchestra Evolution Proposal** (combining approved findings into a unified specification).

> [!NOTE]
> In Phase 4, these outputs are generated natively from strict, machine-readable JSON Schema contracts (`AUDIT_REPORT_SCHEMA.json` and `EVOLUTION_PROPOSAL_SCHEMA.json`). The markdown representations below are generated by the Phase 4C renderer and must precisely reflect the underlying governed JSON records.

## 1. Individual Source Audit

Every evaluated repository must yield an audit document in this exact structure, derived from the `AUDIT_REPORT_SCHEMA.json` instance:

```markdown
# Artificer Source Audit: [Repository Owner]/[Repository Name]

## 1. Source Metadata
- **Repository URL**: [Canonical URL]
- **Commit SHA**: [Reviewed Git Commit Hash]
- **License**: [License Type, e.g., MIT / Apache 2.0]
- **Extraction Date**: [YYYY-MM-DD]
- **Lines of Code Examined**: [Paths and Ranges]

## 2. Executive Summary
[Brief assessment of architectural quality, suitability, and alignment with Orchestra goals.]

## 3. Discovered Patterns
### Finding: [Short Name]
- **Finding ID**: [ID]
- **Pattern**: [Pattern Record Path]
- **Finding**: [Objective observation]
- **Classification**: [REFERENCE_ONLY, etc.]
- **Recommendation**: [Proposed action]
- **Assigned Specialist**: [Cloak | Cipher | Clockwork | etc.]
- **Evidence**: [e.g., STATIC_ANALYSIS, RUNTIME_CONFIRMED_BY_AUTHORIZED_EXTERNAL_VALIDATION]

## 4. IP & Licensing Analysis
- **Detected License**: [License]
- **Attribution Obligation**: [Description of attribution requirements]
- **Compatibility Verdict**: [COMPATIBLE | GOVERNOR_REVIEW_REQUIRED | INCOMPATIBLE]
- **Governor Review Required**: [Yes/No]

## 5. Security & Risk Log
- **External Execution Performed**: [False]
- **Dynamic Risk**: [Adversarial comment instructions, build-script hazards]
```

## 2. Amalgamated Orchestra Evolution Proposal

When merging findings, compile a clean, native proposal document derived from the `EVOLUTION_PROPOSAL_SCHEMA.json` instance:

```markdown
# Orchestra Evolution Proposal: [Topic]

## 1. Objective
[Problem statement and context]

## 2. Summary of Source Audits
- [Audit Report ID] (Commit: [SHA])

## 3. Selected Design Patterns
### Proposed Pattern: [Name]
- **Decision ID**: [Governance Decision ID]
- **Pattern Record**: [Record Path]
- **Evolution Mechanism**: [Conceptual adaptation | Safe rewrite]
- **Target Component**: [Where it fits in Orchestra]
- **Owner Specialist**: [Stated specialist, e.g., Clockwork]

## 4. Verification & Testing Plan
- **Verification Strategy**: [Behavioral tests, mock runs]
- **Adversarial Validation**: [Dagger test cases]

## 5. Governance Hand-off
- **Proposal Status**: [DRAFT | UNDER_REVIEW | APPROVED | BLOCKED]
- **Arbiter Status**: [Status]
- **Governor Status**: [Status]
```
